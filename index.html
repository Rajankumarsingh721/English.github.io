Initialize 
// import into your project
import Navbar from 'navbar.js'

// initialize your target
let myNavbarInit = new Navbar('#myNavbar');

// alternativelly initialize on a UL target
// without the navbar wrapper
let myMenu = document.querySelector('ul.nav');
let myMenuInit = new Navbar(myMenu);

// initialize with options
let myMenuInit = new Navbar(
  'selector',
  {
    breakpoint: 800,
    toggleSiblings: true,
    delay: 600
  }
);

// access an initialization object
let myNavbarInit = document.getElementById('myNavbar').Navbar;

// remove navbar functionality from target
// via accessing the public method
myMenuInit.dispose();

More Tutorials

For various other purposes we've set a couple of additional demos to showcase mixins flexibility:

  • Left Navbar - showcases the SIDE column view of the Navbar layout;
  • Right Navbar - showcases the SIDE column view of the Navbar layout, but this time configured for the RIGHT side of a page; also showcases a simple RTL implementation;
  • Left Menu - showcases the SIDE column view of the menu layout, without the Navbar wrapper; the demo also showcases examples on how to inherit various content style rules;
  • Combined Layout - showcases the combination of SCSS mixins to create styling to support both TOP default view and SIDE column view of the Navbar layout and menus;
  • Bootstrap - showcases the dedicated SCSS mixins to create a seamless integration of the Navbar menu with the original Bootstrap Navbar wrapper and markup;
  • Bootstrap Extended - showcases the full Navbar.js layout design and markup in a Bootstrap styled page.
